Node.js 认识模块化

1. 什么是模块？
  • 所有的功能都是以模块的形式存在的
  • 每一个文件都是一个模块
  • 用户编写的代码都会自动封装到一个模块中
  • 模块之间相互独立，在一个模块中使用另一个模块的功能，就必须引入（require），使用的功能必须是另一个模块暴露（exports）的内容
  • 使用require时，如果模块后缀为 .js .json .node 时，模块后缀可以不写，自动补足顺序为 .js .json .node
暴露数据 
exports.属性或方法 = 值;
引入模块 
require("./模块路径");
2. 模块的组成

所有用户编写的代码都放在模块（函数）文件中

用户自定的模块都封装在一个函数中，该函数提供五个参数，这5个参数在模块文件中可以直接使用。

  1. exports : 暴露对象，可以将模块中的数据或方法暴露给引用的地方
  2. require : 引入模块的函数，用于在一个模块中引入另一个模块，并将子模块暴露的数据赋值给变量
  3. module : 模块对象，包含当前模块的所有信息
  4. __filename : 当前模块的文件名
  5. __dirname : 当前模块所在的路径（目录）
3. 主模块

一个项目只允许有一个主模块，通常命名为：main.js， index.js， app.js， package.json文件中的main声明

主模块是整个项目的启动模块，对项目的其他模块进行统筹

4. 模块分类

  1. 自定义模块 ： 项目自己编写的文件模块

    引入 ： let mod = require('./filename.js');

    • 对于子模块，如果没有暴露数据时，返回空对象"{}"
    • 引入自定义模块务必添加"./"（当前路径）：在nodejs中查找模块默认的路径为"node_modules"目录

  2. 第三方模块

    引入 ： let _ = require('underscore');

    • 第三方程序员或公司开发的模块，需要先安装后使用
    • 安装可以使用nmp包管理工具（npm install {package}）(对于package的支持 ： https://www.npmjs.com/ )

  3. 系统模块 ： nodejs 团队已经开发好的功能模块，直接引入使用即可，比如：fs、http、url、path、...

    引入 ： let fs = require('fs');
